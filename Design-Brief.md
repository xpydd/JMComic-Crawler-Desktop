# JMComic Desktop Design Brief

## 1. 输入来源与证据
- 本轮输入：用户要求参考 shadcn 与 cobbin 风格，全面重构界面设计和交互布局。
- 现有参考：`Product-Spec.md` v0.3.0、`tauri-app/src/index.html`、已有查看详情 / 下载本子 / 下载章节主路径。
- 外部风格线索：shadcn 式组件化 shell、细边框、低圆角、克制按钮；Mobbin / cobbin 式真实应用流、可扫读列表、清晰状态分区。
- 必须继承的现成资产：Tauri 单页入口、设置弹窗、封面预览层、日志追加、批量章节渲染和下载防重复逻辑。

## 2. 目标
- 本轮设计目标：把原单列卡片页面重构为桌面应用工作台，提升首屏信息组织、操作路径清晰度和状态反馈质量。
- 服务产物：`dev-builder`
- 这轮完成定义：主界面具备左侧导航、中间工作区、右侧状态面板；所有既有功能主路径保持可用；界面不再使用 emoji 图标；固定文案中文化，组件占位保持紧凑。

## 3. 方向锁定
- 当前推荐方向：shadcn-inspired utility workspace。
- 为什么是这个方向：JMComic Desktop 是重复使用的下载工具，需要清楚的操作入口、紧凑的信息层级和稳定反馈，而不是内容平台或营销首页。
- 暂不采用的方向：不做暗黑电竞风、不做大面积渐变、不做装饰插画、不引入 React / shadcn 依赖，不把章节列表做成瀑布流内容浏览。

## 4. 设计契约
### Visual Theme & Atmosphere
- 克制、精确、安静，偏桌面生产力工具。
- 页面应像一个可长期使用的控制台，而不是临时 demo。
- 视觉重点放在主任务卡片、结果预览和右侧状态。

### Color Palette & Roles
- App background：`#f6f7f9`，用于桌面外壳。
- Surface：`#ffffff`，用于工作区面板和弹窗。
- Border：`#e2e8f0`，用于 shadcn 式细分割。
- Text primary：`#0f172a`，正文和标题。
- Text muted：`#64748b`，辅助说明和元信息。
- Primary：`#2563eb`，只用于主动作和关键 focus。
- Success：`#16a34a`，成功日志和状态。
- Danger：`#dc2626`，错误日志和失败状态。
- Code / log surface：`#0f172a`，用于日志面板。

### Typography Rules
- 字体使用系统栈：`Inter` 可用时优先，否则 system-ui。
- 主标题 20-24px，面板标题 14-16px，正文 14px，辅助文本 12-13px。
- 不使用负 letter-spacing，不用 viewport 缩放字体。
- 结果详情用结构化字段，不使用长段落堆叠。
- 固定界面文案使用中文；允许保留产品名、ID、路径和来自底层异常的技术文本。

### Component Stylings
- Button：高度不小于 40px，触控目标不小于 44px；主按钮蓝底白字，次按钮白底细边框，不做大号展示按钮。
- Input / textarea：白底、细边框、focus ring、稳定高度，不使用内联样式，章节输入高度保持紧凑。
- Card / panel：8px 圆角以内，细边框，极轻阴影；禁止卡片套卡片，面板 padding 控制在 14px 左右。
- Dialog：居中 modal，宽度 520px 以内，遮罩清晰，操作按钮右对齐。
- Badge / status：小尺寸胶囊标签，只展示状态，不承担主操作。
- Icons：所有按钮与导航图标使用内联 SVG，`aria-hidden="true"`；图标按钮必须有 `aria-label`。

### Layout Principles
- 桌面端：`app-shell` 三栏布局，左侧约 200px 导航，中间主工作区，右侧约 280px 状态面板。
- 平板端：中间工作区 + 右侧状态上下/两栏组合，左侧导航收敛。
- 手机端：单栏，导航变为顶部栏，所有操作垂直排列。
- 结果区和日志区分开；日志在右侧状态面板内，结果在主工作区内。

### Depth & Elevation
- 使用边框建立层级，阴影只给 shell panel、modal 和封面预览。
- Hover 通过 border / background / text color 变化，不使用会导致布局漂移的 scale。

### Do's and Don'ts
- Do: 保持查看、下载、章节、日志四类任务的稳定位置。
- Do: 首屏可看到保存路径状态、处理中状态和主操作入口。
- Do: 加载、错误、成功状态必须同时出现在日志和结果区。
- Do: 使用 SVG 图标和可访问标签。
- Do: 控制组件高度、日志高度和结果区空状态，避免大面积空白。
- Do: 可见固定文案中文化。
- Don't: 不使用 emoji 图标。
- Don't: 不显示英文分区标题，例如 Album、Result、Activity。
- Don't: 不使用大面积紫蓝渐变、装饰光球、营销 hero。
- Don't: 不新增复杂任务队列或功能导航。

### Responsive Behavior
- 1440px：三栏完整显示。
- 1024px：右侧状态面板可位于主区下方或保持窄栏。
- 768px 以下：单栏，按钮组换行但不横向滚动。
- 375px：输入、按钮、章节项文本不得溢出。

### Agent Prompt Guide
- Build a restrained desktop utility shell inspired by shadcn/ui and Mobbin app flows.
- Use native HTML/CSS/JS only; do not introduce a framework.
- Preserve existing IDs and JS function contracts unless tests are updated first.
- Prefer structured panels, fine borders, SVG icons, compact metadata, visible focus states.

## 5. 信息架构
- 页面 / 工作区：主工作台、设置弹窗、封面预览层。
- 页面关系：应用启动进入主工作台；设置和封面预览以 overlay 方式打开。
- 导航方式：左侧导航承载主任务分组和设置入口；主工作区承载操作；右侧状态面板承载反馈。

## 6. 页面拆分
| 页面 | 目标 | 主要内容 | 主操作 |
|---|---|---|---|
| 主工作台 | 完成查询、下载和反馈 | 左侧导航、中间任务卡片、结果区、右侧状态面板 | 查看详情、下载本子、下载章节、批量下载 |
| 设置弹窗 | 设置下载保存路径 | 路径输入、目录选择、保存 / 取消 | 选择目录、保存 |
| 封面预览层 | 放大查看封面 | 图片预览 | 点击遮罩关闭 |

## 7. 关键状态
| 页面 | 默认 | 空状态 | Loading | 错误 | 成功 / 结果 |
|---|---|---|---|---|---|
| 主工作台 | 展示本子 ID、章节输入、结果占位、路径状态和日志 | 未查询时结果区展示空状态 | 全局状态条显示处理中，按钮禁用 | 结果区显示错误摘要，日志追加失败详情 | 结果区展示详情或下载成功消息，日志追加成功 |
| 设置弹窗 | 展示当前保存路径 | 空路径表示默认桌面并下载前确认 | 系统目录选择等待返回 | 选择失败写入日志 | 保存后关闭并更新状态面板 |
| 封面预览层 | 展示封面大图 | 无封面不打开 | 图片加载中保持深色背景 | 图片失败保留 alt | 点击关闭返回 |

## 8. 交互规则
- 主路径：输入本子 ID -> 查看详情 -> 结果区展示详情和章节 -> 下载整本或章节 -> 右侧日志持续反馈。
- 次路径：打开设置 -> 选择目录 -> 保存 -> 右侧状态面板立即更新。
- 高风险交互：下载中必须用忙碌锁阻止重复点击；未配置保存路径时才弹确认。
- 允许局部微调的旋钮：面板间距、右侧面板宽度、章节项密度、日志高度。

## 9. 组件与样式约束
- 主按钮：只用于查看详情、下载本子；放在主任务卡片内。
- 表单：label 与输入必须绑定或在视觉上紧邻，placeholder 不替代 label。
- 卡片 / 列表：顶层 panel 可以分区；不允许卡片套卡片。
- 反馈方式：`#loading` 是全局处理状态；`#logs` 是过程日志；`#result` 是结构化结果。
- 风格边界：原生实现模拟 shadcn 的设计语言，不引入新依赖。

## 10. Handoff 要求
- 交付给设计执行时必须带上：三栏布局、状态矩阵、组件规则和禁用 emoji 约束。
- 交付给开发时必须带上：保持 DOM id、函数名、Tauri invoke 参数和测试合约。
- 必须保真的部分：左侧导航 / 中间工作区 / 右侧状态面板；SVG 图标；下载防重复；批量懒渲染。
- 可接受偏差：具体间距和色值可小幅调整，但不能回到单列 demo 布局。

## 11. 开发影响
- 需要前端重点还原的区域：整体 CSS、HTML 外壳、结果渲染 HTML、日志/状态文案。
- 需要后端 / AI 配合的状态：无后端变更，无 AI。
- 不允许自行脑补的点：搜索、账号、收藏、复杂队列、主题切换。

## 12. 验收标准
- [ ] 页面拆分清晰
- [ ] 主路径状态齐全
- [ ] 设计契约足够稳定
- [ ] 关键组件规则可执行
- [ ] handoff 信息齐全
- [ ] 主界面符合三栏 shell / 响应式单栏要求
- [ ] 不使用 emoji 图标
- [ ] 固定界面文案中文化
- [ ] 组件尺寸和面板占位保持紧凑
- [ ] 可继续进入 `dev-builder` 或 `/check`
