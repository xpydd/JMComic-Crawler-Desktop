# JMComic Desktop Design Brief

## 1. 输入来源与证据
- 本轮输入：用户要求补齐项目基线，当前仓库已有 Tauri 桌面端代码和 Python 核心库
- 现有参考：`tauri-app/src/index.html`、`tauri-app/README.md`、根目录 `README.md`
- 必须继承的现成资产：单页工具台结构、查看详情 / 下载本子 / 下载章节三条操作路径、日志区和结果区

## 2. 目标
- 本轮设计目标：把现有桌面客户端整理成可执行的设计规范，供后续检查和开发补齐状态
- 服务产物：`dev-planner` / `dev-builder`
- 这轮完成定义：页面结构、关键状态、交互规则和组件边界清晰，能直接用于 `/check`

## 3. 方向锁定
- 当前推荐方向：克制的桌面工具台，强调输入、执行、反馈
- 为什么是这个方向：用户目标是快速完成查看和下载，不需要营销页、内容流或复杂导航
- 暂不采用的方向：不做沉浸式首页、不做卡片化内容浏览、不做重视觉品牌包装

## 4. 设计契约
### Visual Theme & Atmosphere
- 工具型、轻量、稳定，信息密度中等
- 视觉重心放在输入和执行反馈，不用大面积装饰元素

### Color Palette & Roles
- 背景色：浅灰，用于区分应用外壳和内容面板
- Surface：白色，用于承载表单、日志和结果
- 主强调色：蓝色，仅用于主要执行按钮
- 辅助色：灰色，用于设置、次级按钮和弱提示
- 状态色：绿色表示成功，红色表示错误，深色背景仅用于日志区

### Typography Rules
- 使用系统字体，优先保证跨平台清晰度
- 主标题 24px 左右，面板文字 13-14px，日志使用等宽字体
- 不使用超大标题，不做营销型排版

### Component Stylings
- 按钮：普通操作按钮保持小尺寸，不抢占表单焦点
- 输入框：全宽，明确 placeholder，保持一致高度
- 面板：用于分组任务，圆角不超过 8px
- 弹窗：只承载设置表单和确认操作
- 日志：深色等宽区域，便于扫描过程信息

### Layout Principles
- 桌面端使用居中最大宽度容器，任务区纵向排列
- 主任务区在上，章节下载和日志在下
- 结果区独立放置，避免和日志混在一起

### Depth & Elevation
- 仅面板和弹窗允许轻微阴影
- 普通按钮和输入框不做夸张海拔

### Do's and Don'ts
- ✅ 保持表单、按钮、日志和结果的固定秩序
- ✅ 操作前校验输入，操作中禁用按钮，操作后明确反馈
- ✅ 弹窗内普通按钮保持克制尺寸
- ❌ 不把设置、章节下载、日志做成抢眼主视觉
- ❌ 不加入大字号宣传语、插画首页或多余导航
- ❌ 不新增复杂主题系统

### Responsive Behavior
- 桌面端保持最大宽度工具台
- 窄屏下按钮组允许换行，输入框保持全宽
- 弹窗宽度不超过视口宽度的 90%

### Agent Prompt Guide
- 后续前端实现必须继承“单页工具台”方向
- 普通操作按钮不得做成大字号、大色块主视觉
- 如新增功能，优先增加在现有任务分组内，除非 Product Spec 明确要求新页面

## 5. 信息架构
- 页面 / 工作区：主工作台、设置弹窗
- 页面关系：应用启动进入主工作台；设置弹窗从主工作台覆盖打开
- 导航方式：无全局导航，使用按钮触发局部操作

## 6. 页面拆分
| 页面 | 目标 | 主要内容 | 主操作 |
|---|---|---|---|
| 主工作台 | 完成查看详情和下载 | 本子 ID、保存目录、章节 ID、日志、结果 | 查看详情、下载本子、下载章节 |
| 设置弹窗 | 配置默认保存目录 | 默认保存目录输入框 | 保存、取消 |

## 7. 关键状态
| 页面 | 默认 | 空状态 | Loading | 错误 | 成功 / 结果 |
|---|---|---|---|---|---|
| 主工作台 | 展示输入和操作按钮 | ID 为空时阻止操作并提示 | 显示处理中并禁用按钮 | 结果区展示错误，日志追加错误 | 结果区展示详情或成功消息 |
| 设置弹窗 | 展示已保存目录 | 空目录表示使用程序默认 | 不需要 | 保存失败时保留弹窗 | 保存后关闭并回填主工作台 |

## 8. 交互规则
- 主路径：输入本子 ID -> 查看详情 -> 下载本子或下载章节 -> 查看结果
- 次路径：打开设置 -> 保存默认目录 -> 回填保存目录输入框
- 高风险交互：下载操作需要确认，避免误触发大量下载
- 允许局部微调的旋钮：按钮间距、面板间距、日志高度、结果区最大高度

## 9. 组件与样式约束
- 主按钮：用于查看详情和下载本子，尺寸保持普通工具按钮级别
- 表单：输入框必须带明确 placeholder，保存目录允许为空
- 卡片 / 列表：面板只做任务分组，不做嵌套卡片
- 反馈方式：日志记录过程，结果区承载最终内容
- 风格边界：工具优先，不做内容平台视觉，不做复杂品牌系统

## 10. Handoff 要求
- 交付给设计执行时必须带上：页面拆分、状态矩阵、组件约束
- 交付给开发时必须带上：输入校验、loading 禁用、错误展示、保存目录持久化规则
- 必须保真的部分：主工作台的任务分组顺序、日志和结果分离、设置弹窗轻量化
- 可接受偏差：颜色细节、间距、按钮文案可微调，但不能改变主路径

## 11. 开发影响
- 需要前端重点还原的区域：输入校验、按钮禁用、日志追加、结果展示
- 需要后端 / AI 配合的状态：后端桥接成功、失败和资源缺失；无 AI 状态
- 不允许自行脑补的点：搜索、登录、收藏夹、批量队列、内置阅读器

- [ ] 设计契约足够稳定
- [ ] 关键组件规则可执行
- [ ] handoff 信息齐全
