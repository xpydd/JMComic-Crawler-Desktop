# JMComic Desktop Product Spec

## 1. 产品概述
JMComic Desktop 是一个基于 Tauri 的跨平台桌面客户端，面向希望用图形界面使用 JMComic 下载能力的个人用户。它把现有 `jmcomic` Python 库封装成无需手动写脚本、无需记命令的桌面工具，第一版重点解决“输入 ID、查看详情、预览封面、选择章节并下载到自定义目录”的主路径。

## 2. 当前目标
- 当前阶段：需求迭代
- 本轮目标：为查看和下载动作增加等待时间记录，在操作日志中按秒展示耗时
- 成功标准：查看详情、下载本子、下载单章节和批量下载章节结束时，日志都能显示本次操作耗时秒数，并保持现有查看 / 下载主路径不回归

## 3. 目标用户
- 核心用户：已经知道 JMComic ID、但不想使用命令行或 Python 脚本的个人用户
- 典型场景：用户拿到本子 ID 或章节 ID 后，希望快速查看详情并下载到本地目录
- 核心痛点：命令行参数、Python 环境、依赖安装和下载配置对普通用户有门槛

## 4. MVP 范围
### 必做
- 输入本子 ID 并查看本子详情、本子封面和章节封面
- 在右上角设置弹窗中选择并持久化下载保存目录
- 下载整个本子到设置中的保存目录
- 输入章节 ID 并下载单个章节
- 从本子详情中选择多个章节并批量下载
- 展示操作日志、加载状态、错误信息和成功结果
- 打包为 macOS 可安装桌面应用

### 不做
- 不做账号登录和收藏夹管理
- 不做搜索、排行、分类浏览
- 不做复杂任务队列、暂停、续传和并发控制界面
- 不做内置阅读器
- 不做 AI 推荐、AI 分类、AI 文案生成或图像分析
- 不承诺 Windows / Linux 第一版已可交付

## 5. 功能需求
- 解析用户输入的本子 ID，调用桥接层获取标题、作者、标签、页数、观看、点赞、评论、章节列表、本子封面和章节封面
- 在详情结果区展示本子封面，章节列表中展示每个章节的封面缩略图
- 允许用户点击本子封面或章节封面，在界面中查看较大预览
- 解析用户输入的本子 ID，调用桥接层下载整个本子
- 解析用户输入的单个章节 ID，调用桥接层下载单个章节
- 允许用户在本子详情中勾选多个章节，并一次性批量下载
- 允许用户手动输入多个章节 ID，使用逗号、空格或换行分隔后批量下载
- 允许用户在右上角设置弹窗中选择下载保存目录，并在下次打开时自动沿用
- 下载本子、单章节和多章节批量下载都使用设置中的保存目录；已配置保存目录时直接下载且不二次确认；未配置保存目录时提示用户确认，并默认下载到用户桌面
- 下载动作执行期间必须阻止重复触发；批量章节下载必须对重复章节 ID 去重
- 在操作开始时禁用按钮并显示处理中状态
- 在操作结束时恢复按钮并展示成功或失败结果
- 在日志区域按时间追加操作过程；查看详情、下载本子、下载单章节、批量下载章节结束时必须按秒记录本次耗时
- 使用 Tauri 命令调用打包后的 `jmcomic-bridge` 可执行文件
- 使用 Python 桥接层复用现有 `jmcomic` 库能力，不重新实现下载逻辑

## 6. 页面与状态
### 页面清单
| 页面 | 用途 | 进入方式 |
|---|---|---|
| 主工作台 | 输入 ID、查看详情、预览封面、下载本子、选择章节、下载章节、查看日志和结果 | 应用启动默认打开 |
| 设置弹窗 | 选择并保存下载保存目录 | 点击主工作台右上设置按钮 |
| 封面预览层 | 放大查看本子封面或章节封面 | 点击详情区中的封面 |

### 关键状态
| 页面 | 默认态 | 空状态 | Loading | 错误态 | 成功 / 结果态 |
|---|---|---|---|---|---|
| 主工作台 | 展示本子 ID、章节 ID 输入框、操作按钮、日志和结果区 | 未输入 ID 时阻止操作并提示输入 | 显示处理中，禁用相关按钮 | 结果区展示错误，日志追加失败信息 | 详情展示结构化信息、封面预览入口和章节选择列表 |
| 设置弹窗 | 展示当前下载保存目录和选择目录按钮 | 目录为空时表示未配置保存目录，下载前需要确认并默认保存到桌面 | 选择目录时等待系统对话框返回 | 保存失败时保留输入并提示 | 保存后关闭弹窗，并让后续下载直接使用新目录 |
| 封面预览层 | 展示被点击的封面大图 | 无封面时不打开预览，显示占位状态 | 图片加载中显示占位 | 图片加载失败显示失败提示 | 图片加载完成后可关闭返回主工作台 |

## 7. UI 布局
### 整体布局
应用采用 shadcn 式桌面工作台 shell：左侧窄导航用于产品标识、主任务入口和设置；中间主工作区承载本子查询、章节输入与结果详情；右侧状态面板承载保存路径、处理状态和操作日志。整体参考 Mobbin / cobbin 的真实产品流密度，优先可扫读、可重复操作和稳定状态反馈，不做营销型首页。

### 区域说明
- 左侧导航：显示产品标识、当前任务分组和设置入口；桌面端固定，窄屏时收敛为顶部栏
- 顶部状态栏：显示当前工作区名称、保存路径状态和全局处理中状态
- 本子操作区：包含本子 ID、查看详情和下载本子操作，作为主任务卡片
- 章节操作区：包含章节 ID 输入、单章节下载和批量下载输入章节操作
- 结果区：展示详情、封面预览入口、章节列表、成功消息或错误消息
- 右侧状态面板：展示保存路径、操作日志和运行提示
- 设置弹窗：只处理下载保存目录，不扩展其他设置
- 封面预览层：展示本子封面或章节封面大图，提供关闭操作

### 视觉与组件风格
- 视觉方向：克制、现代、组件化，使用浅色背景、白色面板、细边框和小阴影，避免单色大渐变和装饰性背景
- 组件风格：参考 shadcn 的 sidebar、button、input、card、dialog、badge 组织方式，但在原生 HTML/CSS 中实现，不引入 React 或组件库
- 图标：界面图标使用内联 SVG，禁止使用 emoji 作为按钮或导航图标
- 文案语言：除产品名、ID、路径、技术错误信息外，界面可见固定文案统一使用中文，不展示英文分区标题
- 交互密度：参考 Mobbin / cobbin 的真实应用模式，控制卡片层级、按钮尺寸和面板占位，让查询、下载、日志都能在首屏被扫描；组件不得为了展示感占用过大空间
- 响应式：桌面端三栏，平板端两栏，手机端单栏；所有点击目标不小于 44px，禁止横向滚动

## 8. 用户流程
### 主路径
1. 用户打开桌面客户端
2. 用户输入本子 ID
3. 用户点击查看详情
4. 系统展示本子详情、本子封面、章节列表和章节封面
5. 用户可以点击封面进行预览
6. 用户选择下载整个本子，或勾选多个章节批量下载，或输入章节 ID 下载单个章节
7. 系统使用设置中的保存目录调用桥接层下载并展示结果；如果用户没有配置保存目录，系统先提示将默认下载到桌面，用户确认后再下载

### 次路径 / 例外路径
1. 用户打开右上角设置弹窗，选择下载保存目录并保存
2. 用户未输入 ID 时点击操作，系统阻止并提示
3. 用户未选择章节时点击批量下载，系统阻止并提示
4. 封面地址为空或加载失败时，系统展示占位或失败状态，不阻断详情查看
5. 桥接层返回错误时，系统在结果区和日志区展示失败原因
6. 打包资源缺失或桥接程序不可执行时，Tauri 命令返回错误

## 10. 技术与约束
- 登录要求：第一版不做登录
- 数据存储：下载保存目录使用浏览器 `localStorage` 本地保存；后续如需要更稳可迁移到 Tauri 本地配置文件
- 外部依赖：Tauri、Rust、PyInstaller、Python `jmcomic` 库、JMComic 网络接口、Tauri 目录选择对话框
- 打包约束：macOS 第一版需要把 `tauri-app/dist/jmcomic-bridge` 作为 Tauri resource 打入应用
- 网络约束：详情和下载依赖目标站点可访问性、域名策略和网络环境
- 图片约束：封面预览依赖远程封面 URL 可访问；加载失败不能影响详情文本展示
- 合规约束：用户需自行确认下载内容的版权、年龄限制和当地法律要求
- 安全约束：保存目录来自用户输入，后续实现必须避免静默覆盖非目标文件

- [ ] 输入有效章节 ID 后，可以调用桥接层下载单个章节
- [ ] 勾选多个章节后，可以一次性批量下载这些章节

- [ ] 操作期间按钮禁用，结束后恢复
