# JMComic Desktop Product Spec

## 1. 产品概述
JMComic Desktop 是一个基于 Tauri 的跨平台桌面客户端，面向希望用图形界面使用 JMComic 下载能力的个人用户。它把现有 `jmcomic` Python 库封装成无需手动写脚本、无需记命令的桌面工具，第一版重点解决“输入 ID、查看详情、下载到本地”的主路径。

## 2. 当前目标
- 当前阶段：补全基线
- 本轮目标：为已有桌面端实现补齐可检查、可规划、可继续开发的产品需求基线
- 成功标准：`Product-Spec.md` 足够支撑 `/check` 对照实现完整度，并能交给 `Design-Brief.md` 与 `DEV-PLAN.md` 继续执行

## 3. 目标用户
- 核心用户：已经知道 JMComic ID、但不想使用命令行或 Python 脚本的个人用户
- 典型场景：用户拿到本子 ID 或章节 ID 后，希望快速查看详情并下载到本地目录
- 核心痛点：命令行参数、Python 环境、依赖安装和下载配置对普通用户有门槛

## 4. MVP 范围
### 必做
- 输入本子 ID 并查看本子详情
- 下载整个本子到默认目录或用户指定目录
- 输入章节 ID 并下载单个章节
- 设置并持久化默认保存目录
- 展示操作日志、加载状态、错误信息和成功结果
- 打包为 macOS 可安装桌面应用

### 不做
- 不做账号登录和收藏夹管理
- 不做搜索、排行、分类浏览
- 不做批量任务队列、暂停、续传和并发控制界面
- 不做内置阅读器
- 不做 AI 推荐、AI 分类、AI 文案生成或图像分析
- 不承诺 Windows / Linux 第一版已可交付

## 5. 功能需求
- 解析用户输入的本子 ID，调用桥接层获取标题、作者、标签、页数、观看、点赞、评论和章节列表
- 解析用户输入的本子 ID，调用桥接层下载整个本子
- 解析用户输入的章节 ID，调用桥接层下载单个章节
- 允许用户填写本次保存目录，留空时使用程序默认目录
- 允许用户在设置弹窗中保存默认保存目录，并在下次打开时自动填入
- 在操作开始时禁用按钮并显示处理中状态
- 在操作结束时恢复按钮并展示成功或失败结果
- 在日志区域按时间追加操作过程
- 使用 Tauri 命令调用打包后的 `jmcomic-bridge` 可执行文件
- 使用 Python 桥接层复用现有 `jmcomic` 库能力，不重新实现下载逻辑

## 6. 页面与状态
### 页面清单
| 页面 | 用途 | 进入方式 |
|---|---|---|
| 主工作台 | 输入 ID、查看详情、下载本子、下载章节、查看日志和结果 | 应用启动默认打开 |
| 设置弹窗 | 配置默认保存目录 | 点击主工作台右上设置按钮 |

### 关键状态
| 页面 | 默认态 | 空状态 | Loading | 错误态 | 成功 / 结果态 |
|---|---|---|---|---|---|
| 主工作台 | 展示本子 ID、保存目录、章节 ID 输入框和操作按钮 | 未输入 ID 时阻止操作并提示输入 | 显示处理中，禁用所有按钮 | 结果区展示错误，日志追加失败信息 | 详情展示结构化信息，下载展示成功消息 |
| 设置弹窗 | 展示当前默认保存目录 | 目录为空时表示使用程序默认目录 | 不需要异步 loading | 保存失败时保留输入并提示，当前版本仅本地存储 | 保存后关闭弹窗并回填主工作台 |

## 7. UI 布局
### 整体布局
应用采用单页工具台结构：顶部标题与设置入口，中间按任务分组，底部显示日志与结果。第一版优先清晰、克制、低学习成本，不做营销型首页。

### 区域说明
- 顶部区域：显示产品标题和设置入口
- 本子操作区：包含本子 ID、保存目录、查看详情和下载本子操作
- 章节操作区：包含章节 ID 输入和下载章节操作
- 日志区：展示按时间追加的操作记录
- 结果区：展示详情、成功消息或错误消息
- 设置弹窗：只处理默认保存目录，不扩展其他设置

## 8. 用户流程
### 主路径
1. 用户打开桌面客户端
2. 用户输入本子 ID
3. 用户点击查看详情
4. 系统展示本子详情和章节列表
5. 用户选择下载整个本子，或复制章节 ID 下载单个章节
6. 系统调用桥接层下载并展示结果

### 次路径 / 例外路径
1. 用户打开设置弹窗，填写默认保存目录并保存
2. 用户未输入 ID 时点击操作，系统阻止并提示
3. 桥接层返回错误时，系统在结果区和日志区展示失败原因
4. 打包资源缺失或桥接程序不可执行时，Tauri 命令返回错误

## 9. AI 能力需求
| 能力 | 用途 | 输入 | 输出 |
|---|---|---|---|
| 无 | 第一版不接入 AI，避免把下载工具复杂化 | 无 | 无 |

## 10. 技术与约束
- 登录要求：第一版不做登录
- 数据存储：默认保存目录使用浏览器 `localStorage` 本地保存
- 外部依赖：Tauri、Rust、PyInstaller、Python `jmcomic` 库、JMComic 网络接口
- 打包约束：macOS 第一版需要把 `tauri-app/dist/jmcomic-bridge` 作为 Tauri resource 打入应用
- 网络约束：详情和下载依赖目标站点可访问性、域名策略和网络环境
- 合规约束：用户需自行确认下载内容的版权、年龄限制和当地法律要求
- 安全约束：保存目录来自用户输入，后续实现必须避免静默覆盖非目标文件

## 11. 验收标准
- [ ] 应用启动后默认进入主工作台
- [ ] 未输入本子 ID 时，查看详情和下载本子不会调用后端
- [ ] 输入有效本子 ID 后，可以展示标题、作者、标签、页数、互动数据和章节列表
- [ ] 输入有效本子 ID 后，可以调用桥接层下载整个本子
- [ ] 输入有效章节 ID 后，可以调用桥接层下载单个章节
- [ ] 设置默认保存目录后，主工作台输入框立即回填，并在刷新后保留
- [ ] 操作期间按钮禁用，结束后恢复
- [ ] 成功和失败都能在日志区与结果区看到明确反馈
- [ ] macOS 构建脚本能产出可安装包
